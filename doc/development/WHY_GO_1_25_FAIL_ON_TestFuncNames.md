# Why TestFuncNames Fails on go1.25

## The Error

```
=== RUN   TestFuncNames
    func_names_test.go:63: version: go1.25.10
    func_names_test.go:104: expect func name: ...func_names.init.func1, actual: ...func_names.init.func5
--- FAIL: TestFuncNames (0.00s)
```

The test `test/xgo_test/func_names/func_names_test.go` validates Go's function naming via `runtime.FuncForPC`. It defines a top-level closure `var c = func(){}` and expects it to be named `init.func1`. On go1.25, it's `init.func5` — shifted by 4.

## Root Cause

### The compiler-generated autogen file

xgo instruments test packages via `RegisterFuncTab()` (`instrument/instrument_reg/instrument_reg.go:13`), which writes modified source files into the overlay. For each file, it injects:

1. **4 stub template closures** — `__xgo_register_N`, `__xgo_trap_N`, `__xgo_trap_var_N`, `__xgo_trap_varptr_N`. These are initialized with template functions that reference runtime dispatch functions (`runtime.XgoRegister`, `runtime.XgoTrap`, etc.). They are NOT init closures themselves.

2. **Multiple `func init()` blocks** — at least 4 synthetic init functions generated by the registration code (e.g., `func init(){ __xgo_func_info_1_0.Func=F; ... }` and `func init(){ __xgo_init_1(); ... }`). These ARE compiled as `init.func1` through `init.func4`.

3. **Function trap wrappers** — each tracked function gets a wrapper that calls `__xgo_trap_N` for interception.

### Why go1.24 is unaffected

On go1.24, `needLocalRuntime` is `false` (the go1.25 GOMODCACHE overlay restriction doesn't apply). xgo runs `TestFuncNames` with a plain `go test` — no overlay, no `compiler_extra.json`, no `XGO_COMPILER_SYNTAX_REWRITE_ENABLE`. The patched compiler's `AfterFilesParsed` is never invoked. The top-level closure remains `init.func1`.

Log comparison:

| Aspect | go1.24 | go1.25 |
|--------|--------|--------|
| `needLocalRuntime` | `false` | `true` |
| `XGO_COMPILER_SYNTAX_REWRITE_ENABLE` | not set | `true` |
| overlay | none | `.xgo/gen/overlay.json` |
| packages instrumented | 0 | 537 |
| autogen stubs | none | `__xgo_register_0` etc. (4 closures) |
| `init.func` numbering | `func1` | `func5` |

### Why full instrumentation is triggered on go1.25

`needLocalRuntime` (`cmd/xgo/main.go:710`) is `true` for `goVersion.Minor >= 25`. This forces `importRuntimeDepGenOverlay()` to:
1. Copy the xgo runtime to a local directory (`.xgo/gen/modules/`)
2. Modify the `go.mod` with `replace` directives
3. Add blank imports (`import _ "runtime/trace"`) to Go source files via overlay
4. Set up `compiler_extra.json` with the package mapping

This in turn triggers the full instrumentation pipeline for all loaded packages, including the `func_names` test package.

## The Fix

The test already uses version-gated constants for closure names (`closure_name_go1.22_test.go` vs `closure_name_go1.17_18_19_20_21_test.go`). The same pattern is applied for go1.25:

- **`closure_name_go1.22_test.go`**: `//go:build go1.22 && !go1.25` → `init.func1`
- **`closure_name_go1.25_test.go`**: `//go:build go1.25` → `init.func5`
- **`closure_name_go1.17_18_19_20_21_test.go`**: `//go:build go1.17 && !go1.22` → unchanged
